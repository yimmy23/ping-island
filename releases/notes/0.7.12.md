## 亮点
- `0.7.12` 聚焦高频会话更新下的 Island 响应速度，减少会话状态频繁刷新时的 UI 抖动和额外计算。
- 浮动宠物的电量/用量提示现在优先展示当前窗口剩余额度，悬停时不需要打开完整配额面板也能快速确认余量。

## 修复
- 修复 detached 浮动气泡贴近窗口边缘时可能被裁切的问题，气泡四周会保留轻量安全边距。
- 降低多会话、高频 hook 更新场景下的重复发布和列表派生开销，让菜单栏 Island 更稳定。
- 调整浮动用量 footer 的悬停文案与展示范围，避免在小空间里挤入过重的信息层级。

## 说明
- 本次发布不改变 hook、远程连接或更新通道语义。
- Sparkle build 号同步提升到 `26`，用于确保客户端更新判断单调递增。

## Highlights
- `0.7.12` focuses on Island responsiveness under frequent session updates, reducing UI churn and avoidable derived-state work.
- The floating pet usage hint now prioritizes the current-window remaining quota, so hover gives a quick answer without opening the full quota panel.

## Fixes
- Fixed detached floating bubbles being clipped when rendered close to the window edge by preserving a small gutter around the bubble.
- Reduced duplicate publishing and list-derivation overhead when multiple hook-driven sessions update frequently.
- Tuned floating usage footer hover copy and scope so compact detached panels stay readable.

## Notes
- This release does not change hook behavior, remote connection semantics, or the update channel.
- The Sparkle build number is now `26` so client update ordering remains monotonic.
