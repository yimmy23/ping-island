## 亮点
- 支持让 Buddy 离开岛单独工作，浮动宠物的拖拽、通知气泡、展开内容和首次引导现在更完整、更自然。
- 新增 Claude / Codex 用量展示，可在展开态 Island、离岛浮动气泡和活跃消息气泡里直接查看剩余额度与摘要。
- 已有用户升级到 `0.5.1` 时，离岛 Buddy 的迁移提示会按版本更可靠地触发一次，减少首次升级后错过引导的情况。
- 对用量徽标、AskUserQuestion 选项卡片、拖拽稳定性和提醒消失时机做了一轮集中打磨，整体交互更顺手。

## 修复
- 修复浮动宠物通知与展开气泡在部分场景下的样式、状态衔接和高度预留不一致问题。
- 稳定 detached 气泡与共享会话状态之间的切换，减少悬停预览、拖拽和提醒状态互相抢占的情况。
- 改进首次展示 upgrade hint 的时机，在启动初期 notch 暂时不可展示时也会补发，避免提示被静默吞掉。

## 说明
- `0.5.1` 是一次围绕 Buddy 离岛工作、Claude / Codex 用量可见性与升级体验稳定性的正式发布，建议直接覆盖安装 `0.4.1` 或 `0.5.0`。
- 本次发布汇总了 `v0.4.1` 之后已合入 `main` 的离岛宠物交互、用量摘要、onboarding 与升级提示可靠性改进。

## Highlights
- Buddy can now work off-island with a more complete floating-pet flow, including dragging, bubble notifications, expanded content, and first-run guidance.
- Added Claude / Codex usage visibility so you can check remaining quota and summaries from expanded Island panels, floating bubbles, and active-message bubbles.
- Upgrading users now get the off-island Buddy migration hint once per shipped version more reliably, so the guidance is less likely to be missed during startup.
- Shipped a broader UX polish pass across usage badges, AskUserQuestion cards, drag behavior, and missed-attention dismissal timing.

## Fixes
- Fixed mismatches in styling, state handoff, and reserved footer space for floating pet notifications and expanded bubbles.
- Stabilized detached-bubble state transitions so hover previews, dragging, and attention notifications are less likely to fight each other.
- Improved upgrade-hint scheduling so the migration prompt can retry after transient startup states block the first docked-notch presentation.

## Notes
- `0.5.1` is the formal release for the off-island Buddy workflow, Claude / Codex usage visibility, and upgrade-experience stability work. It is a direct upgrade from `0.4.1` or `0.5.0`.
- This release rolls up the detached-pet interaction, usage-summary, onboarding, and upgrade-guidance reliability work merged into `main` after `v0.4.1`.
