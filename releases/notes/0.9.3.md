## 亮点
- 初步加入 CodeBuddy CLI 兼容支持，并将可回答的 AskUserQuestion 流程接到 Island 内联表单与 CodeBuddy 兼容的 hook 响应格式。
- Homebrew Cask 发布自动化现在会随稳定版 GitHub Release 同步外部 tap，让 `brew install --cask erha19/tap/ping-island` 使用正式发布资产。

## 修复
- 修复并发 hook 事件覆盖当前可见状态后，旧的审批或提问 socket 可能悬空、导致客户端一直等待的问题。
- 避免 Codex App 后台标题生成辅助会话被误识别成真实提醒或重新绑定到最近的 Codex 会话。
- 避免 Qoder CLI 集成提示在设置面板刷新时反复出现。

## 隐私与诊断
- 诊断导出中的 Claude hook debug 日志改为有界、脱敏摘要，保留支持排查需要的事件形状、哈希标识与安全元数据，避免导出原始 JSONL 负载。

## 说明
- Sparkle build 号提升到 `33`，版本号提升到 `0.9.3`。
- CodeBuddy CLI 支持仍是阶段性兼容，IDE 与 CLI hook 语义继续保持独立。

## Highlights
- Added initial CodeBuddy CLI compatibility, including answerable AskUserQuestion flows through Island inline forms and the CodeBuddy-compatible hook response shape.
- Homebrew Cask release automation now syncs the external tap from stable GitHub Releases, so `brew install --cask erha19/tap/ping-island` installs the official release assets.

## Fixes
- Fixed stale approval or question sockets that could remain blocked after concurrent hook events replaced the currently visible intervention.
- Prevented Codex App background title-generation helper sessions from appearing as real reminders or rebinding onto recent Codex sessions.
- Stopped the Qoder CLI integration notice from reappearing every time the settings panel refreshes.

## Privacy and Diagnostics
- Claude hook debug logs in diagnostic exports are now bounded, redacted summaries that keep support-useful event shape, hashed identifiers, and safe metadata without exporting raw JSONL payloads.

## Notes
- The Sparkle build number is now `33`, and the app version is now `0.9.3`.
- CodeBuddy CLI support is still partial compatibility; IDE and CLI hook semantics remain independent.
