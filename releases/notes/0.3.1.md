## 亮点
- 改进远程桥接会话恢复逻辑，应用重启后会更稳地保留远端桥接生命周期，减少需要重新接管 SSH 工作流的情况。
- 优化远程 Claude 提问展示方式，让远程问题保持可见但不阻塞当前会话流转。
- 支持为 SSH Remote 主机直接配置显式端口，避免再依赖目标地址上的额外拼接技巧。

## 修复
- 修复远程主机尚未安装 Hermes 时的引导脆弱性，首次接入和补装流程更顺滑。
- 继续打磨 OpenClaw 形象，让激活态和整体吉祥物序列里的辨识度更稳定。

## 说明
- `0.3.1` 是一次基于最新 `main` 的维护发布，重点补强远程桥接稳定性、SSH 连接配置体验和客户端细节打磨。

## Highlights
- Improves remote-bridge session recovery so app restarts preserve the bridge lifecycle more reliably and reduce the need to reattach SSH workflows manually.
- Keeps remote Claude questions visible without blocking the active session flow.
- Adds explicit SSH port support for remote hosts so connections no longer depend on target-string hacks.

## Fixes
- Hardens remote bootstrap when Hermes is not installed on the host yet, making first-run and recovery flows smoother.
- Continues polishing the OpenClaw mascot so its active-state silhouette stays more legible across the lineup.

## Notes
- `0.3.1` is a maintenance release from the latest `main`, focused on remote-bridge resilience, SSH connection ergonomics, and a final round of client polish.
