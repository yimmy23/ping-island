## 亮点
- `0.8.0` 支持最新版 Qoder CLI `0.2.5+`，会以 Claude Code 兼容 hook 流写入 `~/.qoder/settings.json`，并保留用户已有的无关设置。
- Qoder CLI 终端会话现在优先保留真实 terminal / tmux / IDE-hosted terminal 上下文，点击 Ping Island 会回到实际运行中的 Qoder CLI 窗口，提供与 Claude Code 一致的监控和回跳体验。
- AskUserQuestion 的问题和选项展示经过重构，长选项不再被截断，通知打开时会优先进入可操作问答卡片。

## 修复
- 重构 Qoder 体系消息通知路由，避免历史聊天内容抢占待处理审批或问题卡片。
- 修复 AskUserQuestion 选项在 hook 事件切换和通知展示中丢失完整文案的问题。
- 修复 cmux / Ghostty 系终端会话从 Ping Island 激活时可能无法拉起正确窗口的问题。
- 修复 detached 浮动气泡淡出测试的稳定性问题。
- 修复通用 `PingIslandBridge` 打包和 Hermes 插件 manifest 名称问题。

## 说明
- 本次发布覆盖 Qoder CLI、AskUserQuestion、通知路由、终端回跳和打包稳定性，重点是让 Claude 兼容 hook 客户端获得一致的监控体验。
- 本地 Hooks 配置现在提供明确的批量移除入口；该操作仍限定在本地 hook 配置文件范围内，不影响 IDE extension 或远程 bridge 卸载流程。
- Sparkle build 号同步提升到 `28`，用于确保重新发布的 `0.8.0` 客户端更新判断单调递增。

## Highlights
- `0.8.0` supports Qoder CLI `0.2.5+` by writing Claude Code-compatible hooks into `~/.qoder/settings.json` while preserving unrelated user settings.
- Terminal-hosted Qoder CLI sessions now keep their real terminal / tmux / IDE-hosted terminal context, so clicking Ping Island returns to the actual Qoder CLI window with a Claude Code-like monitoring and focus experience.
- AskUserQuestion presentation and interaction have been reworked so long option copy is preserved, and notification openings route directly to the actionable question card.

## Fixes
- Reworked Qoder-family notification routing so stale chat content no longer wins over pending approval or question cards.
- Fixed AskUserQuestion option copy being truncated or lost across hook event transitions and notification surfaces.
- Fixed cmux / Ghostty-family session activation from Ping Island so terminal windows are raised more reliably.
- Stabilized the detached floating bubble fade test.
- Fixed universal `PingIslandBridge` packaging and the Hermes plugin manifest name.

## Notes
- This release covers Qoder CLI, AskUserQuestion, notification routing, terminal focus, and packaging stability, with a focus on consistent monitoring for Claude-compatible hook clients.
- Local Hooks configuration now has an explicit bulk removal action; it remains limited to local hook config files and does not remove IDE extensions or remote bridges.
- The Sparkle build number is now `28` so the republished `0.8.0` remains monotonic for client update ordering.
