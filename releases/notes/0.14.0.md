## 亮点
- 首次启动会引导用户完成 Island hooks 权限演示，设置页也可以重新播放，帮助新用户理解审批、回答和会话列表的关系。

- 修复 Codex `Stop` hook 被当成空闲状态的问题；Codex 会话在后台 agent 仍运行时会继续保持等待输入/处理中语义。

- 完成 onboarding 演示后会自动清理合成的 Ping Island Demo 会话，避免演示数据残留在主列表。

## Highlights
- First launch now guides users through an Island hooks permission walkthrough, and Settings can replay it so new users can see how approvals, answers, and the session list fit together.
- App Store builds keep hook configuration writes behind explicit user authorization, avoiding silent failures or container-directory writes in sandboxed builds.
- Chat transcript updates are smoother: message ordering, Markdown rendering, and the processing spinner avoid broad list rebuilds during long conversations and streaming updates.

## Fixes
- Fixed Codex `Stop` hooks being treated as idle; Codex sessions now keep their waiting-for-input / in-progress semantics while background agents are still running.
- Fixed floating pet bubbles being clipped at transparent-window edges, losing rounded corners under child content, or nudging the mascot when previews and notifications collapse.
- Floating completion notifications now size from rendered content, so short replies no longer leave excessive empty space while longer replies still respect the existing screen cap.
- The onboarding demo now archives its synthetic Ping Island Demo session after completion so demo data does not linger in the primary list.

## Notes
- The Sparkle build number is now `42`, and the app version is now `0.14.0`.
- This is an experience and reliability release on top of `0.13.1`, focused on first-run hooks guidance, App Store sandbox behavior, Codex state, chat transcript stability, and floating pet bubble rendering.
