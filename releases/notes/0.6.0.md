## 亮点
- `0.6.0` 让你可以在设置里关闭后台自动检查更新，同时保留手动检查更新入口，减少不想被打扰时的后台轮询。
- Codex Desktop 线程发现现在增加了更稳妥的兜底刷新路径，即使 app-server 长时间不主动推送新线程事件，Ping Island 也更容易继续发现并跟上新的本地会话。
- 远程 SSH Linux 主机的 bridge 发布资产补齐了 `aarch64` 包，使用 arm64 服务器时不再只能依赖 `x86_64` 发行物。

## 修复
- 修复无法关闭后台自动检查更新的问题，对应 issue `#103`。
- 修复部分机器上 Codex Desktop 新线程不再持续被发现、`session-associations.json` 停滞的问题，对应 issue `#101`。
- 修复 release 流程未上传 Linux `aarch64` remote bridge 资产的问题，对应 issue `#90`。

## 说明
- `0.6.0` 是在 `0.5.3` 之后发布的下一个客户端小版本，建议当前用户直接覆盖安装。
- 这次版本聚焦在更新体验、Codex 集成可靠性，以及 SSH 远程 arm64 支持补齐，不额外扩大产品面。

## 关联项
- `#103` 希望增加关闭自动检查更新的能力
- `#101` Bug: Ping Island 无法持续发现新的 Codex Desktop 线程
- `#90` 希望增加 ssh aarch64 的支持

## Highlights
- `0.6.0` adds a settings toggle to disable background update checks while keeping manual update checks available when you want them.
- Codex Desktop thread discovery now has a more reliable fallback refresh path, so Ping Island can keep finding newer local threads even when the app-server stops pushing live thread events.
- The remote SSH Linux bridge release now includes an `aarch64` asset, so arm64 hosts are no longer forced onto an `x86_64`-only release package.

## Fixes
- Fixed the inability to turn off background update checks, matching issue `#103`.
- Fixed a case where newer Codex Desktop threads stopped being discovered and `session-associations.json` stalled, matching issue `#101`.
- Fixed the release flow so Linux `aarch64` remote bridge assets ship with the client release, matching issue `#90`.

## Notes
- `0.6.0` is the next minor client release after `0.5.3`, and existing users should upgrade in place.
- This release focuses on update behavior, Codex integration reliability, and SSH arm64 release coverage without expanding the broader product surface.
