## 亮点
- Claude Code 审批流更稳定：当新版 Claude Code 的 `PermissionRequest` 没有携带 `tool_use_id` 时，Ping Island 会复用前一个工具事件缓存到的 tool id，让创建文件、Bash 等审批可以继续和正确的工具调用绑定。

## 修复

- 为缺失 `tool_use_id` 的 Claude approval 路径补充回归测试，覆盖 approval 清理状态。

## Highlights
- Made Claude Code approvals more reliable. When newer Claude Code `PermissionRequest` payloads omit `tool_use_id`, Ping Island now reuses the cached tool id from the preceding tool event so file creation, Bash, and similar approvals stay bound to the correct tool call.

## Fixes
- Fixed stale approval interventions that could remain after approving a Claude Code file creation or command request and interfere with later approval prompts.
- Added regression coverage for Claude approval cleanup when the approval payload itself lacks `tool_use_id`.

## Notes
- The Sparkle build number is now `31`, and the app version is now `0.9.1`.
- Related issues: #136, #137, #138.
