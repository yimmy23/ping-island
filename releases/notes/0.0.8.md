## 亮点
- 改进在线 release notes 的加载回退逻辑，更新说明窗口在远端 Markdown 不可用时也能更稳定地显示版本说明。
- 补强更新链路的单元测试覆盖，明确校验 GitHub appcast、404 更新源和离线网络等常见异常场景。

## 说明
- 当 Sparkle 条目没有可直接读取的完整 Markdown 说明时，Ping Island 现在会更稳妥地回退到可用说明源，减少“发现更新但说明为空”的情况。
- 这一版是一次小型补丁发布，主要聚焦自动更新体验与版本说明展示稳定性。

## Highlights
- Improved the release-notes fallback path so the update window can still show version notes more reliably when the remote Markdown source is unavailable.
- Expanded unit-test coverage around the update flow, including GitHub appcast handling and common failure cases such as 404 feeds and offline networking.

## Notes
- When a Sparkle item does not expose a directly readable full Markdown note, Ping Island now falls back more safely to the next available source instead of leaving the update UI without useful notes.
- This is a small patch release focused on auto-update reliability and release-notes presentation stability.
