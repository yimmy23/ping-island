## Fixes
- Fixed historical Codex completion notifications replaying after session refreshes, window rebuilds, or docked / detached transitions. Completion popups now require recent real activity.
- Fixed Codex memories / maintenance threads being treated as normal sessions and auto-opening the Island panel. Hook, app-server, and rollout fallback ingress paths now suppress this auxiliary traffic.
- Fixed AskUserQuestion custom replies after choosing "Other" so the input receives a real caret, first-responder focus, and IME support.

## Notes
- The Sparkle build number is now `60`, and the app version is now `0.24.0`.
- This release is based on latest `main` after `v0.23.1` and focuses on Codex notification noise reduction, auxiliary thread filtering, and question reply input reliability.
