## Highlights
- Added a separate Mac App Store build lane with the `PingIslandAppStore` target, dedicated Info.plist, sandbox entitlements, build script, and submission notes while keeping the direct Developer ID / Sparkle lane isolated.
- Supported `AskUserQuestion` flows can now accept custom typed answers instead of limiting users to preset choices, matching issue `#139`.
- The closed Island can now expose Claude / Codex usage headroom, making context and quota pressure visible without opening the panel.
- Experimental settings now require a deliberate unlock before they appear, keeping in-progress features out of the normal settings path.
- Added App Store privacy policy links to the README files, with privacy documentation covering diagnostics, updates, remote bridge behavior, and local configuration access.

## Notes
- The Sparkle build number is now `34`, and the app version is now `0.10.0`.
- The App Store lane remains separate distribution preparation; direct downloads and Sparkle updates still use the regular `PingIsland` target.
