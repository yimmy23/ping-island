## 亮点
- `0.7.0` 适配了新版 Codex CLI 的权限请求协议，Codex 在等待 `PermissionRequest` 时会继续在 Island 内展示批准/拒绝入口，避免 CLI 长时间卡在 hook 阶段。
- Codex rollout 兜底解析现在可以识别 `request_user_input` 提问，让只出现在 rollout 日志里的计划模式/输入请求也能浮到 Island。
- 会话列表改进了子 Agent 的归属展示，将明确挂靠在主 Agent 下的子会话收拢到主会话下方，列表更容易扫读。
- 独立宠物形态的用量提示更克制：详细额度信息留在气泡里，只有额度低于 60% 时才在宠物旁显示轻量提醒。

## 修复
- 修复新版 Codex CLI 在 `PermissionRequest hook` 阶段阻塞、Island 内没有权限交互的问题，对应 issue `#110`。
- 修复 Codex `request_user_input` 提问只出现在 rollout fallback 中时不会生成 Island 输入卡片的问题。
- 修复终端承载的 Codex 会话被误导向 Codex App 打开路径的问题，并保持 Codex 身份与终端聚焦目标分离。
- 保留用户已有的 Claude `statusLine` 配置，仅在字段缺失或已由 Ping Island 管理时刷新托管命令，对应 issue `#111`。
- 移除历史上未实际参与当前 bridge 流程的 `island-state.py` hook 安装路径，降低 hook 配置噪音和维护误导。

## 说明
- `0.7.0` 是 `0.6.0` 之后的下一个客户端版本，建议 Codex CLI 用户优先升级。
- 本次发布聚焦 Codex 权限/输入可靠性、Claude hook 配置安全性、子 Agent 展示，以及独立宠物用量提示，不额外扩大远程或发布流程范围。

## Highlights
- `0.7.0` adapts Ping Island to the newer Codex CLI permission request protocol, so Codex approval/deny actions continue to surface in Island instead of leaving the CLI stuck in the hook phase.
- Codex rollout fallback parsing now recognizes `request_user_input` prompts, so plan-mode questions and input requests that only appear in rollout logs can still become Island cards.
- The session list now groups explicit subagents under their parent agent, making busy multi-agent runs easier to scan.
- Detached pet usage indicators are quieter: detailed quota windows stay inside the bubble, while the small pet-side hint appears only when remaining quota drops below 60%.

## Fixes
- Fixed Codex CLI sessions getting stuck at `PermissionRequest hook` without an Island approval UI, matching issue `#110`.
- Fixed Codex `request_user_input` prompts being missed when they only appeared through rollout fallback parsing.
- Fixed terminal-hosted Codex sessions being routed like native Codex App sessions, while keeping agent identity separate from terminal focus targets.
- Preserved existing user Claude `statusLine` settings unless the field is missing or already managed by Ping Island, matching issue `#111`.
- Removed the unused legacy `island-state.py` hook installation path so managed hook ownership matches the current bridge flow.

## Notes
- `0.7.0` is the next client release after `0.6.0`, and Codex CLI users should upgrade first.
- This release focuses on Codex permission/input reliability, Claude hook configuration safety, subagent display, and detached pet usage presentation without expanding the remote or release surfaces.
