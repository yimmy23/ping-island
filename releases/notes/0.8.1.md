## 修复
- 修复 Codex hooks 监听可能失效的问题：当 Xcode 调试版和已安装版 Ping Island 同时运行时，后启动实例会抢占 `/tmp/island.sock`，导致 Codex hook 事件送到另一个进程。
- 调试启动现在默认仍会执行单实例保护，确保 hook bridge、Codex app-server 轮询和当前可见菜单栏实例保持一致。

## 说明
- 如确实需要并行调试多个 Ping Island 实例，可显式设置 `PING_ISLAND_ALLOW_MULTIPLE_INSTANCES=1`。
- Sparkle build 号提升到 `29`，版本号提升到 `0.8.1`。

## Fixes
- Fixed Codex hooks appearing to stop listening when an Xcode debug build and the installed Ping Island app were running at the same time. The later instance could take over `/tmp/island.sock`, sending Codex hook events to a different process.
- Debug launches now keep single-instance enforcement enabled by default, so hook bridge delivery, Codex app-server polling, and the visible menu bar instance stay aligned.

## Notes
- Multiple local instances remain available for explicit debugging by setting `PING_ISLAND_ALLOW_MULTIPLE_INSTANCES=1`.
- The Sparkle build number is now `29`, and the app version is now `0.8.1`.
